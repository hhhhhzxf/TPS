# TPS 后端完成计划

> 文档说明：本文件记录后端待补全能力、优先级与落地计划。


**版本：** v1.1  
**日期：** 2026-05-02  
**范围：** `backend/` Spring Boot 后端  
**目标：** 支撑 Android 手机端稳定完成二手交易核心链路

---

## 1. 当前状态结论

当前后端已经具备主要功能骨架，但还不适合直接作为生产后端使用。

综合判断：

| 维度 | 状态 | 说明 |
|------|------|------|
| 功能骨架 | 基本完成 | 认证、商品、订单、消息、管理接口已有 |
| Android 联调 | 基本具备 | 已统一公开地址约定、响应结构，剩余真机联调验证 |
| 生产安全 | 基本具备 | 已补接口限流、健康检查、迁移与日志，剩余部署侧安全加固 |
| 交易可靠性 | 不完善 | 订单并发、防重复购买、状态机还需增强 |
| 文档契约 | 基本具备 | 已有接口契约、错误码、联调说明，仍需随代码持续同步 |
| 自动化验证 | 已有基础覆盖 | 已增加 H2 + MockMvc 集成测试，并补充健康检查验证 |

预估完成度：**约 88%**。

## 4. 后端完成阶段

### Phase B1：认证与权限加固

目标：保证 Android 登录稳定、安全、可续期。

任务：

- 增加 refresh token。
- 统一认证失败、token 过期、权限不足错误码。
- 封禁用户 token 立即失效或请求被拒。
- 已完成：主要业务 Controller 改为使用 Spring Security Principal / `@AuthenticationPrincipal`。
- 管理员接口只允许 `ADMIN` 访问。

验收：

- 普通用户不能访问 `/api/admin/**`。
- 被封禁用户不能继续调用受保护接口。
- token 过期后 APP 能根据错误码触发刷新或重新登录。

### Phase B2：Android API 契约稳定

目标：让 Gemini / Android 端可以稳定接接口。

任务：

- 固定响应结构：`code`、`message`、`data`。
- 固定分页结构字段。
- 为每个接口补充请求示例、响应示例。
- 明确图片 URL、WebSocket URL、BASE_URL。
- 输出 Swagger / OpenAPI。

验收：

- APP 可基于 `api-contract.md` 建 Retrofit DTO。
- 接口字段变更必须先改文档再改代码。

### Phase B3：商品、图片、收藏

目标：支撑 Android 首页、详情、发布商品、收藏。

任务：

- 商品列表支持分页、分类、关键词、价格区间、成色。
- 商品详情返回卖家摘要、图片完整 URL、收藏状态。
- 图片上传只允许图片类型。
- 图片上传返回完整 URL。
- 收藏接口增加明确 `POST`、`DELETE`、`GET status`。

验收：

- Android 真机能浏览商品图片。
- 发布商品后能在首页看到。
- 收藏状态在列表和详情页一致。

### Phase B4：消息与 WebSocket

目标：支撑买卖双方安全聊天。

任务：

- WebSocket 握手携带 token。
- 服务端从认证身份确定 senderId。
- 历史消息查询校验会话成员。
- 会话列表返回对方信息、商品摘要、未读数。
- 消息支持分页加载。

验收：

- 用户不能伪造其他用户发消息。
- 用户不能读取不属于自己的会话。
- Android 真机可连接 WebSocket 并收发消息。

### Phase B5：订单交易闭环

目标：稳定完成购买、支付、发货、收货。

任务：

- 创建订单时防止重复购买。
- 支付后锁定商品或订单。
- 状态流转只允许合法路径。
- 发货增加物流单号字段。
- 订单详情返回商品摘要、买家摘要、卖家摘要。
- 订单状态变更生成通知。

验收：

- 同一商品不能被多个买家同时完成交易。
- 买家不能替卖家发货，卖家不能替买家确认收货。
- APP 订单页能完整展示买入 / 卖出订单。

### Phase B6：评价、举报、退款、管理

目标：补齐平台运营闭环。

任务：

- 完成评价接口。
- 完成举报商品接口。
- 完成退款申请与处理接口。
- 管理员处理举报、下架商品、处理纠纷。
- 管理员统计返回清晰 DTO。

验收：

- 完成订单后才可评价。
- 同一用户同一订单只能评价一次。
- 管理员可处理举报并通知用户。

### Phase B7：生产准备

目标：后端可部署、可排查、可维护。

任务：

- 使用 `dev`、`prod` profile。
- 数据库、Redis、JWT、上传路径全部改环境变量。
- 已完成：增加健康检查。
- 已完成：增加基础接口限流。
- 增加 smoke test。
- 补充部署说明。

验收：

- 新环境按文档可启动。
- smoke test 可验证主链路。
- 不再提交生产密码和固定密钥。

## 5. Claude Code 执行建议

推荐按以下顺序派发后端代码任务：

1. 认证与 Principal 重构。
2. WebSocket 鉴权和消息越权修复。
3. 图片上传校验与完整 URL。
4. 订单并发与状态机加固。
5. 评价 / 举报 / 退款接口补齐。
6. DTO 与错误码统一。
7. 测试和生产配置。

每次任务必须输出：

- 修改文件列表
- 新增 / 修改接口
- 数据库字段变化
- Android 端需要同步的 DTO 字段
- 验证命令或验证步骤

## 6. 完成定义

后端达到“可交付给 Android 真机使用”需要满足：

- Android 手机可访问后端 API。
- Android 手机可加载商品图片。
- Android 手机可建立 WebSocket 聊天。
- 普通用户主链路可走通：注册 → 登录 → 浏览 → 发布 → 收藏 → 聊天 → 下单 → 支付 → 发货 → 收货 → 评价。
- 管理员主链路可走通：登录 → 查看统计 → 用户管理 → 商品举报处理 → 订单查看 → 公告发布。
- 所有接口有文档、错误码和基础测试。
