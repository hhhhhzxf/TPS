# TPS 生产就绪清单

**版本：** v1.0  
**日期：** 2026-04-28  
**范围：** 后端服务、Android APP 联调、生产部署准备

---

## 1. 生产目标

上线前后端需要满足：

- Android 手机可稳定访问。
- 核心交易链路可用。
- 基础安全问题已修复。
- 配置不泄露敏感信息。
- 有日志、健康检查和回归验证。
- 出现问题可以回滚和排查。

---

## 2. 环境配置

| 配置项 | 开发环境 | 生产要求 |
|--------|----------|----------|
| 数据库地址 | 本地 MySQL | 环境变量注入 |
| 数据库密码 | 可本地配置 | 禁止提交仓库 |
| Redis 地址 | 本地 Redis | 环境变量注入 |
| JWT secret | 开发固定值 | 强随机密钥，环境变量注入 |
| 文件目录 | `./img` | 独立挂载目录或对象存储 |
| API 协议 | HTTP 可接受 | HTTPS |
| WebSocket | WS 可接受 | WSS |
| 日志级别 | DEBUG | INFO / WARN |

---

## 3. 安全清单

- [ ] JWT secret 不写死在 `application.yml`。
- [ ] 数据库用户名和密码不写死。
- [ ] 登录、验证码、上传接口有限流。
- [ ] 管理员接口有 `ADMIN` 权限。
- [ ] 用户不能访问他人订单。
- [ ] 用户不能访问他人聊天记录。
- [ ] WebSocket 不能伪造 `senderId`。
- [ ] 上传文件必须校验类型和大小。
- [ ] 生产环境关闭详细异常堆栈返回。
- [ ] CORS / allowed origins 不使用无限制 `*`。

---

## 4. 数据库清单

- [ ] 建议引入 Flyway / Liquibase。
- [ ] 生产环境不使用 `ddl-auto:update`。
- [ ] 关键表有必要索引。
- [ ] 收藏表有用户 + 商品唯一约束。
- [ ] 订单表有商品有效订单约束或业务锁。
- [ ] 消息表按会话和时间可高效查询。
- [ ] 数据库字符集使用 `utf8mb4`。
- [ ] 有备份和恢复方案。

---

## 5. 后端接口验收

| 链路 | 验收项 |
|------|--------|
| 认证 | 注册、登录、刷新 token、封禁用户拦截 |
| 商品 | 发布、列表、搜索、详情、编辑、下架 |
| 图片 | 上传、访问、Android 真机展示 |
| 收藏 | 收藏、取消收藏、收藏状态一致 |
| 消息 | 会话、历史消息、WebSocket 收发、权限校验 |
| 订单 | 创建、支付、发货、确认、取消、并发防护 |
| 评价 | 完成订单后评价、不可重复评价 |
| 举报 | 用户举报、管理员处理 |
| 通知 | 消息、订单、公告通知 |
| 管理 | 用户封禁、商品下架、统计、订单查看 |

## 6. Android 真机验收

- [ ] 真机能访问 API。
- [ ] 真机能登录并保存 token。
- [ ] 真机能加载商品图片。

## 8. 部署建议

推荐生产部署结构：

```text
Android APP
    |
HTTPS / WSS
    |
Nginx
    |
Spring Boot Backend
    |
MySQL + Redis + Object Storage
```

Nginx 负责：

- HTTPS 证书
- API 反向代理
- WebSocket 反向代理
- 静态图片代理或对象存储转发

## 10. 回滚方案

上线前需要准备：

- 后端旧版本包。
- 数据库备份。
- 配置备份。
- 图片目录备份或对象存储保留策略。
- Nginx 配置回滚方案。

任何涉及数据库结构的变更，都必须明确是否可逆。
